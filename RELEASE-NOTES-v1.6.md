## ⭐ ตัวใหญ่ที่สุด: หัวข้อเหนือทุกตัวเลือกในเกม (DiceHeader)

**ขาดไป 389 จาก 398 รายการ — เกือบทั้งหมด**

นี่คือข้อความที่ขึ้นเหนือกล่องตัวเลือกทุกครั้งที่คุณต้องตัดสินใจ ที่ผ่านมาส่วนใหญ่**โผล่เป็นจีนดิบ**

| ก่อน | หลัง |
|---|---|
| 你的抉择 | การตัดสินใจของเจ้า |
| 由谁出战 | ให้ผู้ใดออกรบ |
| 你的对手 | คู่ต่อสู้ของเจ้า |
| 你的反应 | ปฏิกิริยาของเจ้า |
| 赌博胜负 | ผลแพ้ชนะการพนัน |

ตอนนี้ **ครบ 398/398 = 100%** และแปลรวดเดียวทั้งชุด เพื่อให้รูปแบบที่ซ้ำกัน
(`X的想法` / `X的决定` / `X的反应`) ออกมาเป็นแบบเดียวกันทั้งเกม

## เก็บอะไรอีกบ้าง

- **บทเนื้อเรื่อง 267 จุด** — รวมบทตะโกนตัวใหญ่ในฉากต่อสู้, คำอธิบายท่าไม้ตาย/อาวุธลับ/เสียงด่า
  และบรรทัด **"เงื่อนไขปลดล็อก: เงิน>=500"** ที่ก่อนหน้านี้เป็นจีนล้วน
- **แก้ 2 บรรทัดที่ยังเป็นอังกฤษ** ตกค้างจากม็อดอังกฤษต้นทาง
- **แก้ 1 บรรทัดที่พังมาตลอด** — บรรทัดที่มีโค้ดสี `<color=#FF7979>` เขียนผิดรูปแบบ
  ทำให้ระบบตัดข้อความผิดตำแหน่ง **entry นั้นไม่เคยทำงานเลยตั้งแต่ต้น** ตอนนี้ใช้ได้แล้ว

## 🎯 แก้ปัญหา "ข้อความตอนสู้ปนภาษามั่ว" ที่ค้างมานาน

อาการที่เคยเจอ: บันทึกการต่อสู้ (ช่องข้อความด้านข้างตอนดวล) มีไทย+จีน+อังกฤษปนกันมั่วขึ้นเรื่อย ๆ ทุกรอบ

พอแกะโค้ดเกมออกมาก็เห็นสาเหตุชัด ๆ:

```csharp
logText.text = logText.text + log + "\n";   // ต่อท้ายข้อความเดิมไปเรื่อย ๆ
```

ตัวแปลอัตโนมัติเลยอ่าน**ข้อความทั้งก้อนที่ยาวขึ้นทุกรอบ** แล้วแปลซ้ำทับของเดิม → ปนกันมั่ว
และตัวข้อความยัง**ฝังอยู่ในโค้ดเกมเป็นจีนตัวเต็ม** ไม่ผ่านระบบแปลปกติ จึงไม่มีทางแก้ด้วยไฟล์คำแปล

v1.6 ตั้งค่าบอกตัวแปลตรง ๆ ว่าช่องนี้เป็น "ช่องบันทึก" ให้แปลเฉพาะบรรทัดใหม่:

```ini
GameLogTextPaths=/[UI]/MainUI/Layer_2/GameLog/Viewport/Content
```

ตำแหน่งนี้อ่านมาจากโครงสร้างฉากในไฟล์เกมโดยตรง แล้วยืนยันซ้ำกับ log ของเครื่องจริง

> **ข้อนี้ยังไม่ได้ทดสอบในเกม** — ถ้าเปิดหน้าต่างบันทึกการต่อสู้แล้วยังปนอยู่ รบกวนแจ้งด้วย

## 🔍 คุณภาพ: สำนวนแยกเพศถูกตั้งแต่แรก

v1.5 ทำระบบที่รู้ว่า **ใครพูดบรรทัดไหน** (จากสคริปต์บทของตัวเกม) รอบนี้เอามาใช้ตั้งแต่ขั้นแปล
ผู้แปลทุกคนเห็นชื่อ+เพศของคนพูดควบคู่ไปกับบทเลย

ผลตรวจ 977 บรรทัด: **ผิดแค่ 1 บรรทัด** (ตัวละครชายถูกให้พูดคำลงท้ายหญิง) แก้ก่อนใส่ลงไฟล์แล้ว

v1.3 กับ v1.5 ต้องไล่แก้เพศย้อนหลัง 185 และ 111 บรรทัด — **v1.6 ไม่ต้องแก้ย้อนหลังเลย**

## ✅ ตรวจสอบแล้ว

- ความครอบคลุมคำแปล **97.22% → 97.95%** (จากทุก key ที่เกมมีจริง)
- บรรทัดเดิมทั้ง 85,841 บรรทัดไม่ถูกแตะ ยกเว้น 3 บรรทัดที่ตั้งใจแก้
- **ไม่มี key ซ้ำเพิ่มขึ้นเลย** (0 รายการ)
- UTF-8 ไม่มี BOM · ไม่มี CRLF · ทั้งเกม `ครับ`=0 · `ค่ะ` เดี่ยว=0 · `เธอ`=0

### เจอบั๊กในเครื่องมือตรวจของเราเอง (บันทึกไว้กันพลาดซ้ำ)

รอบแรกเครื่องมือตัดคำผิด ทำให้ **420 บรรทัดที่แปลไปแล้วดูเหมือนยังไม่ได้แปล** และเกือบใส่ซ้ำลงไฟล์
จับได้ตอนตรวจ key ซ้ำ ก่อนจะปล่อยออกมา — ย้อนกลับแล้วทำใหม่ทั้งหมด
(ตัวเลขความครอบคลุมจริงจึงเป็น 97.22% ไม่ใช่ 96.4% ที่เคยรายงานไว้)
